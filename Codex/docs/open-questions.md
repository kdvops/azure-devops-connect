# Open Questions

Estas preguntas no bloquean el diseño inicial:

1. ¿La rama debe crearse para todos los issue types?
2. ¿Qué tipos deben excluirse?
3. ¿Quién puede crear manualmente una rama?
4. ¿Debe existir un repositorio por proyecto o varios?
5. ¿Debe permitirse elegir repositorio desde el issue?
6. ¿Cuál será la longitud máxima predeterminada?
7. ¿Debe soportarse draft issue?
8. ¿Debe agregarse comentario Jira al crear la rama?
9. ¿Debe almacenarse el correo del autor del commit?
10. ¿Debe existir retención para OperationLog?
11. ¿PAT será aceptable en producción interna?
12. ¿Se publicará eventualmente en Marketplace?
