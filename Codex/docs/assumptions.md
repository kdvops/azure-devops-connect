# Assumptions

- El primer objetivo es Jira Cloud.
- Azure DevOps es Azure DevOps Services.
- Los repositorios son Azure Repos Git.
- El MVP es interno.
- Se permite PAT temporalmente.
- Forge Storage es suficiente para el MVP.
- Una instalación Jira conecta inicialmente con una organización Azure DevOps.
- Cada proyecto Jira tiene un repositorio predeterminado.
- La rama no se renombra cuando cambia el summary.
- La issue key permanece estable.
- La rama base es configurable.
- La UI no llama Azure DevOps directamente.
