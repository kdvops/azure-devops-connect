# Acceptance Criteria

El MVP se acepta cuando:

1. Un administrador configura Azure DevOps.
2. Un administrador mapea proyecto Jira a repositorio.
3. Se configura rama base.
4. La creación de issue crea una rama.
5. La operación es idempotente.
6. El panel muestra la rama.
7. Se verifica si la rama existe.
8. Se consulta el último commit.
9. Se abre la rama en Azure DevOps.
10. Puede crearse manualmente.
11. Puede reintentarse.
12. Los errores no exponen secretos.
13. Hay pruebas unitarias.
14. Hay pruebas de integración.
15. Hay pruebas de contrato.
16. Hay documentación operativa.
17. `npm run build` finaliza correctamente.
18. `npm run lint` finaliza correctamente.
19. `npm test` finaliza correctamente.
20. `forge lint` finaliza correctamente.
